# Phase 6 — Plan 01: App Scaffold, Auth Flow, WebSocket Client, Live Connections Table

## Goal

Stand up the Next.js 14 Pages Router scaffold with dark Bootstrap 5 theme, implement JWT login with first-run setup handling, build the `useWebSocket` hook with exponential backoff reconnect and a connection status indicator pill, and render a live connections table that updates via DOM mutation (no JSX re-render) on every WebSocket push.

After this plan, a user can open `http://localhost:3000`, log in, and watch live connection rows appear in a table within 1 second of capture events.

```ts
export function countryFlag(code: string | null | undefined): string {
  if (!code || code.length !== 2) return '—';
  const offset = 127397;
  return String.fromCodePoint(
    code.toUpperCase().charCodeAt(0) + offset,
    code.toUpperCase().charCodeAt(1) + offset,
  );
}
```

## Wave 2: Auth Hook and Login Page

### Task 2-A: useAuth Hook

**`frontend/hooks/useAuth.ts`:**

```ts
import { useState, useEffect, useCallback } from 'react';
import { apiLogin, apiSetup } from '../lib/api';

const TOKEN_KEY = 'pnpg_token';

export function useAuth() {
  const [token, setToken] = useState<string | null>(null);
  const [needsSetup, setNeedsSetup] = useState<boolean>(false);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    // Restore session token if present. Setup state is discovered lazily on the first login
    // attempt: POST /api/v1/auth/login returns HTTP 503 with { detail: "Run setup first" }
    // when no password has been configured. /api/v1/health does NOT expose a needs_setup flag
    // (it only returns status/probe/db/stream — pnpg/api/routes/status.py:44-50).
    // The dashboard always shows the login form first; the setup branch is only entered after
    // the 503 response is received.
    const stored = sessionStorage.getItem(TOKEN_KEY);
    setToken(stored);
  }, [];

  const login = useCallback(async (password: string) => {
    setError(null);
    setLoading(true);
    try {
      const res = await apiLogin(password);
      // Backend returns HTTP 503 + { detail: "Run setup first" } when no password is configured.
      // This is the ONLY signal for first-run state; /api/v1/health does not expose it.
      if (res.detail === 'Run setup first') {
        setNeedsSetup(true);
        setLoading(false);
        return;
      }
      const tok = res?.data?.access_token;
      if (!tok) {
        setError(res?.detail ?? 'Login failed');
        setLoading(false);
        return;
      }
      sessionStorage.setItem(TOKEN_KEY, tok);
      setToken(tok);
    } catch {
      setError('Network error — is the backend running?');
    } finally {
      setLoading(false);
    }
  }, []);

  const setup = useCallback(async (password: string) => {
    setError(null);
    try {
      const res = await apiSetup(password);
      if (res?.data?.message) {
        setNeedsSetup(false);
        await login(password);
      } else {
        setError(res?.detail ?? 'Setup failed');
      }
    } catch {
      setError('Network error');
    }
  }, [login]);

  const logout = useCallback(() => {
    sessionStorage.removeItem(TOKEN_KEY);
    setToken(null);
  }, []);

  return { token, needsSetup, loading, error, login, setup, logout };
}
```

## Wave 3: WebSocket Hook and Status Indicator

### Task 3-A: useWebSocket Hook (UI-10)

**`frontend/hooks/useWebSocket.ts`:**

Exact implementation matching research Pattern 2.

Key contract points (from VALIDATION.md):
- WS URL: `ws://127.0.0.1:8001/api/v1/ws/live?token=${token}`
- Only connect if `token` is truthy
- On `message`: parse JSON; if `type === 'batch'`, call `onBatch(msg.events)`; if `type === 'heartbeat'`, ignore
- On `close`: schedule reconnect with `Math.min(30000, 1000 * 2 ** attemptRef.current)` delay; increment attempt
- On `open`: reset `attemptRef.current = 0`; set status `'connected'`
- On `error`: call `ws.close()` (triggers `onclose` → backoff)
- On cleanup (unmount): `wsRef.current?.close()`
- Expose: `{ status: 'connecting' | 'connected' | 'disconnected', ws: wsRef }`

```ts
// Return shape:
{ status: WsStatus; ws: React.MutableRefObject<WebSocket | null> }
```
