## Wave 1: AlertsPanel Component (UI-02, UI-08)

### Task 1-A: AlertsPanel

**`frontend/components/AlertsPanel.tsx`**

**Ownership decision (resolves Issue 6):**  
- `AlertsPanel` is **display-only** — it receives `alerts` as a prop and calls the API for actions only (suppress/resolve). It never fetches alerts itself.
- `index.tsx` owns the `alerts` state. It populates it via two sources:
  1. **Initial load:** `GET /api/v1/alerts?status=active` on mount (after login), so the panel is not empty before the first WS push.
  2. **Live updates:** WS batch events — each event in `events` array may have an `alerts` array that is appended to state.
- This is explicit and unambiguous — AlertsPanel's loading/error/empty states refer to the *action* (suppress/resolve) state, not data fetching.

Props:
```tsx
interface Props {
  alerts: AlertEvent[];
  isInitialLoading: boolean;  // true while index.tsx is still fetching initial alerts
  initialError: string | null;  // error from initial GET /api/v1/alerts
  token: string;
  onAlertActioned: (alertId: string) => void;
}
```

Layout: scrollable card list, max-height to avoid overflow, newest alert at top.

Each alert card has:
- Left border 4px solid colored by severity (`var(--sev-warning)` / `var(--sev-alert)` / `var(--sev-critical)`)
- Bootstrap `badge` for severity using `SEVERITY_BADGE[alert.severity]`
- `rule_id`, `process_name`, `reason` text
- `dst_hostname ?? dst_ip` if present
- Timestamp `toLocaleTimeString()`
- `[Suppress]` btn-outline-warning, `[Resolve]` btn-outline-success
- Spinner + disabled buttons while actioning

Action handler calls `apiPatchAlert(token, alertId, action)` then calls `onAlertActioned(alertId)` on success.

States:
- `isInitialLoading=true` → skeleton cards (from parent's initial fetch)
- `initialError` → `alert-danger` with message (from parent's initial fetch)
- `alerts.length === 0` and not loading → "No active alerts ✓"
- Action in progress → spinner in that card's buttons
- Action error → inline `text-danger` badge beside the button

### Task 1-B: Wire AlertsPanel into pages/index.tsx

**Initial alert load (assigned to index.tsx — explicit ownership):**
```ts
const [alerts, setAlerts] = useState<AlertEvent[]>([]);
const [alertsLoading, setAlertsLoading] = useState(false);
const [alertsError, setAlertsError] = useState<string | null>(null);

// On mount (after token is available):
useEffect(() => {
  if (!token) return;
  setAlertsLoading(true);
  apiAlerts(token, 'active')
    .then(res => setAlerts(res?.data ?? []))
    .catch(() => setAlertsError('Failed to load alerts'))
    .finally(() => setAlertsLoading(false));
}, [token]);
```

**Live WS update — append to existing state:**
```ts
function processEvents(events: any[]) {
  const newConns: ConnectionEvent[] = [];
  const newAlerts: AlertEvent[] = [];
  for (const e of events) {
    newConns.push(...(e.connections ?? []));
    newAlerts.push(...(e.alerts ?? []));
  }
  tableRef.current?.injectRows(newConns);
  if (newAlerts.length > 0) {
    setAlerts(prev => [...newAlerts, ...prev].slice(0, 200));
  }
}
```

Layout — split main content:
```
│ AlertsPanel (col-4) │ ConnectionsTable (col-8) │
```

Pass `isInitialLoading={alertsLoading}` and `initialError={alertsError}` to `<AlertsPanel>`.

Add `onAlertActioned` handler that removes by `alert_id` from state.

## Wave 2: Pause/Resume Toggle (UI-07)

### Task 2-A: Add Pause/Resume to useWebSocket

Additions to `hooks/useWebSocket.ts`:
```ts
const isPausedRef = useRef(false);
const pendingRef = useRef<any[]>([]);
const [isPaused, setIsPaused] = useState(false);

// In ws.onmessage: if paused, push to pendingRef; else call onBatch

function pause() { isPausedRef.current = true; setIsPaused(true); }
function resume() {
  isPausedRef.current = false;
  setIsPaused(false);
  if (pendingRef.current.length > 0) {
    onBatch(pendingRef.current);
    pendingRef.current = [];
  }
}

// Return: { status, ws: wsRef, isPaused, pause, resume }
```

**Pause button in navbar:**
- Paused → `btn-warning` "▶ Resume"
- Running → `btn-outline-secondary` "⏸ Pause"

**Paused banner below navbar:**
```tsx
{isPaused && (
  <div className="alert alert-warning m-0 rounded-0 text-center py-1" role="status">
    ⚠ Updates paused — {pendingCount} events buffered.
    <button className="btn btn-sm btn-warning ms-2" onClick={resume}>Resume</button>
  </div>
)}
```
