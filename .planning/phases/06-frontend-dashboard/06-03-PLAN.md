## Wave 1: Charts (UI-03, UI-04)

### Task 1-A: ConnectionsPerApp BarChart

**`frontend/components/ConnectionsPerApp.tsx`**

Wrapped in `dynamic(() => import(...), { ssr: false })` at usage site (Pitfall 4 from research).

Props:
```tsx
interface Props { connections: ConnectionEvent[]; }
```

Derive data inside the component (no separate hook needed):
```ts
const data = useMemo(() => {
  const counts: Record<string, number> = {};
  for (const c of connections) {
    counts[c.process_name] = (counts[c.process_name] ?? 0) + 1;
  }
  return Object.entries(counts)
    .map(([name, count]) => ({ name, count }))
    .sort((a, b) => b.count - a.count)
    .slice(0, 10);  // top 10 processes
}, [connections]);
```

Recharts: `BarChart` with `Bar` fill `#58a6ff`, dark axes, dark tooltip background, `ResponsiveContainer width="100%" height={200}`.

Empty state: `<p className="text-muted text-center">No connection data yet</p>`

**Import guard** вҖ” the component file exports the chart directly. The page wraps with `dynamic`:
```tsx

  () => import('../components/ConnectionsPerApp'),
  { ssr: false }
);
```

### Task 1-B: ConnectionsPerSecond LineChart (rolling 60s)

**`frontend/components/ConnectionsPerSecond.tsx`**

Props:
```tsx
interface Props { batchCount: number; }
// batchCount: number of connections in the latest WS batch (passed from parent)
```

Internal state:
```ts
const bufferRef = useRef<{ ts: number; count: number }[]>([]);
const [chartData, setChartData] = useState<{ ts: number; count: number }[]>([]);
```

On `batchCount` prop change: push `{ ts: Date.now(), count: batchCount }` to `bufferRef`.

1Hz `setInterval` updates `chartData` from buffer and trims entries older than 60s:
```ts
useEffect(() => {
  const id = setInterval(() => {
    const cutoff = Date.now() - 60_000;
    bufferRef.current = bufferRef.current.filter(e => e.ts >= cutoff);
    setChartData([...bufferRef.current]);
  }, 1000);
  return () => clearInterval(id);
}, []);
```

Recharts `LineChart` with `isAnimationActive={false}` (mandatory вҖ” prevents visual jitter).

XAxis: `new Date(ts).toLocaleTimeString()` from each data point.

## Wave 2: CaptureStatus Indicator (UI-05)

### Task 2-A: CaptureStatus Component

**`frontend/components/CaptureStatus.tsx`**

Props: `{ token: string }`

Polls `GET /api/v1/status` every 5 seconds via `setInterval` in `useEffect`.

**Backend contract (locked):** `/api/v1/status` returns `{ capture: "running", interface, uptime, probe_type }` **always while the backend is alive** (`pnpg/api/routes/status.py:24`). The field `capture` is hardcoded to `"running"` вҖ” it does not reflect a real start/stop state. There is **no `capture: "stopped"` response from a live backend.

Rendering states:
- Poll succeeds вҶ’ `bg-success` badge: "в—Ҹ Active (\{probe_type\})" + uptime in small muted text
- Poll fails (network error / backend down) вҶ’ `bg-danger` badge: "вң• Backend unreachable" + italic retry message
- First poll pending вҶ’ muted spinner

```tsx
// State machine (three states only):
type StatusState = 'loading' | 'active' | 'unreachable';
```

Also shows `probe_type` beside the badge when active.

> **Do not render a "Stopped" state.** If the backend goes down, the fetch throws a network error вҖ” handle it as `'unreachable'`, not `'stopped'`.

Wire into navbar in `pages/index.tsx`:
```tsx
<CaptureStatus token={token} />
```
